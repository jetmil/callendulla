# Callendulla

> *Календарь-нянька, который заставляет тебя цвести.*

Telegram-бот, который помнит твои задачи и пинает в нужном тоне — от
мягкого стёба до собранно-жёсткого. Несколько голосовых характеров.
Голосовой дневник. Ночные часы тишины. Самоэскалация при игноре.

**Self-host. Open source. Ничего не уходит в чужое облако кроме того
LLM-провайдера, который ты сам подключил.**

> Проект в ранней публичной стадии. Часть фич в README помечена как
> **WIP** — это значит «реализовано в задумке, ещё не написано в код
> этого репозитория». Двигаюсь итеративно, чтобы каждый кусок выходил
> с тестами и без сюрпризов.

---

## Что внутри

- Календарь событий с напоминаниями (one-shot и повторяющиеся через
  rrule)
- Шесть профилей речи: от тёплого/нейтрального до жёсткого с матом.
  Шесть архетипов с собственным голосом, словарём идиом и жанром
  шуток — **WIP**
- Четыре ступени эскалации внутри любого профиля (от стёба к
  собранно-жёсткому)
- Inline-кнопки на каждом пинке: ✅ сделал · 💤 +1 час · 🌅 завтра ·
  🔇 тихо на 12 часов
- Quiet hours 22–09 локального времени юзера — ночью триггеры не палят,
  переносятся на утро
- Cap-эскалация — если триггер дошёл до потолочного тона и 3+ итераций
  без реакции, автоматический snooze 12 часов + сброс тона (не
  превращается в бесконечный мат каждый час)
- iCal-фид для подключения календаря в Google/Apple/Outlook — **WIP**
- Голосовой дневник: записал voice в Telegram → расшифровка → сохранил
  с шифрованием at-rest — **WIP**
- Multi-user: owner + members в рамках одной семьи/команды, разные
  профили речи у каждого
- REST API на FastAPI для веб-интеграций

## Быстрый старт (self-host)

Нужно: Linux/macOS, Docker 24+, 2 GB RAM. Доменное имя с TLS —
опционально (бот работает и без), нужно только если планируешь
веб-интеграции и iCal-фид наружу.

```bash
git clone https://github.com/jetmil/callendulla
cd callendulla
cp .env.example .env

# Открой .env, заполни минимум:
#   TELEGRAM_BOT_TOKEN  — получи у @BotFather в Telegram
#   OWNER_TG_ID         — твой user_id, узнать через @userinfobot
#   LLM_PROVIDER + LLM_API_KEY — gemini / openai / anthropic / ollama
#   SECRET_KEY          — openssl rand -hex 32
#   DIARY_ENCRYPTION_KEY — см. комментарий в .env.example
#   POSTGRES_PASSWORD   — любой непустой пароль

docker compose up -d
```

Контейнер при старте сам подождёт PostgreSQL и применит миграции
(entrypoint.sh → alembic upgrade head). Никакой отдельной команды.

Найди своего бота в Telegram (имя задал в @BotFather), пиши `/start`.
Если `OWNER_TG_ID` совпадает с твоим — получишь роль owner и доступ ко
всем командам.

Полная установка с TLS, webhook и nginx-прокси — `docs/install.md`
(*готовится*).

## Архитектура

Три процесса в одном Docker-образе, управляются через `supervisord`:

- **api** (FastAPI на 8290) — REST + будущая веб-консоль
- **bot** (aiogram polling по умолчанию, webhook опционально) — Telegram
- **scheduler** (APScheduler) — тикает раз в минуту, обрабатывает
  триггеры

Стек: PostgreSQL 16 (хранение), Redis 7 (кеш + rate-limit), Alembic
(миграции), Loguru (логи), pytest для тестов.

LLM: bring your own key. Поддерживаются Gemini, OpenAI, Anthropic, а
также локальный Ollama (если есть GPU или хотя бы 16 GB RAM).

TTS: Piper (полностью локально) либо Microsoft edge-tts (бесплатный,
облачный, без API-ключа). Опционально CosyVoice3 для премиум-голосов.
Pre-rendered банки для коротких фраз + on-the-fly генерация для
уникальных пинков.

## Безопасность

Перед публичным релизом проверено и закрыто:

- Cross-user изоляция в кеше LLM-ответов (NudgeCache использует
  плейсхолдеры вместо реальных названий событий)
- iCal-токены — random per-user, отдельная команда ротации
- Webhook signature для Telegram (если включаешь webhook-режим)
- Rate-limit на регистрацию и на LLM-запросы per user
- Шифрование голосового дневника at-rest через Fernet
- BYOK для LLM — твои данные не уходят в облака, на которые ты не
  подписался сам

Полный hardening checklist для операторов — в `SECURITY.md`.

Нашёл уязвимость? **НЕ открывай Issue.** Пиши на jetmil@proton.me.

## Зачем

Три года искал работающий вариант: трекеры задач — скучные таблицы,
AI-ассистенты — корпоративные и без характера, Telegram-боты — игрушки.
Сделал то, что хотел: бот, который знает меня, шутит, давит когда надо,
и затыкается ночью.

Выкатываю под AGPL потому что хочу, чтобы код жил дольше, чем мой
текущий VPS, и чтобы крупный игрок не мог просто завернуть его в свой
SaaS и не вернуть ни строчки доработок.

Это не корпоративный SaaS, а инструмент для тех, кто умеет поднять
docker compose и не боится своего VPS.
